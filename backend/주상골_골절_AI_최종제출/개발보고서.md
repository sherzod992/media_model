## 1. 프로젝트 개요

### 1.1 배경

주상골(舟狀骨, Scaphoid)은 손목을 구성하는 8개의 수근골 중 하나로, 낙상 시 손을 짚는 동작에서 가장 빈번하게 골절이 발생한다. 주상골 골절은 X-ray 영상에서 골절선이 매우 미세하게 나타나 임상 현장에서도 초기 진단을 놓치는 경우가 많으며, 적시에 치료받지 못할 경우 무혈성 괴사(Avascular Necrosis)로 이어질 수 있어 임상적으로 매우 위험하다.

본 프로젝트는 딥러닝 기반 이미지 분류 모델을 활용하여 주상골 X-ray 영상에서 골절 여부를 자동으로 판독하는 AI 시스템을 개발하고, 임상 적용 가능한 수준의 성능을 달성하는 것을 목표로 한다.

### 1.2 성능 목표

| 지표 | 목표값 | 의미 |
|------|:------:|------|
| 민감도 (Sensitivity) | ≥ 98% | 골절 환자를 놓치지 않음 (TP / TP+FN) |
| 특이도 (Specificity) | ≥ 85% | 정상을 불필요하게 의심하지 않음 (TN / TN+FP) |
| AUC | ≥ 0.98 | 분류 성능 종합 지표 |
| F1-score | ≥ 0.92 | 정밀도와 재현율의 조화평균 |

> **임상적 우선순위:** False Negative(골절을 정상으로 오판)는 임상적으로 가장 위험한 오류이므로 민감도를 최우선으로 최적화한다.

### 1.3 평가 단위

- **이미지 단위:** 55장의 개별 X-ray 이미지 기준 평가
- **환자·측면 단위(케이스 단위):** 동일 환자·동일 방향의 여러 이미지를 하나의 케이스로 묶어 최대 확률값(MAX)으로 평가 — 임상 판단과 동일한 단위

## 3. 데이터셋

### 3.1 원본 데이터셋

본원 수집 주상골 X-ray 영상으로 구성된다.

| 구분 | 골절 (Fracture) | 정상 (Normal) | 합계 |
|------|:--------------:|:------------:|:----:|
| Train | 223장 | 172장 | 395장 |
| Val | 25장 | 22장 | 47장 |
| Test | 35장 | 20장 | 55장 |
| **합계** | **283장** | **214장** | **497장** |

- 총 **76명의 환자** 데이터
- 1명의 환자가 여러 방향(AP, Lateral, Oblique 등)의 X-ray를 포함
- **정상(Normal) 레이블 = 골절 환자의 반대 손** (실제 건강인 X-ray가 아님)

#### 데이터 분할 원칙

데이터 누수(Data Leakage) 방지를 위해 **환자 단위(Patient-level)** 로 분할하였다. 동일 환자의 여러 방향 영상이 학습/검증/테스트 세트에 혼재될 경우 모델이 환자 특유의 골격 형태를 학습하여 성능이 과대 평가될 수 있다.

### 3.2 정상 데이터의 구조적 한계 발견

초기 실험에서 모델 특이도가 50~75% 수준에 머무르는 원인을 분석한 결과, 다음의 구조적 문제를 발견하였다.

- **모든 정상 이미지 = 골절 환자의 반대 손 X-ray**
- 따라서 모델은 "실제로 건강한 손목의 X-ray"를 한 번도 학습하지 못한 상태
- 정상 이미지들도 골절 환자의 신체에서 나온 것이어서 미묘한 골격 특징을 공유함
- 결과적으로 정상 판별 능력(특이도)에 구조적 상한선이 존재

### 3.3 MURA v1.1 외부 데이터 추가

위 구조적 한계를 해결하기 위해 Stanford 대학이 공개한 **MURA(Musculoskeletal Radiographs) v1.1** 데이터셋을 활용하였다.

- **출처:** Stanford ML Group — MURA v1.1 (공개 의료 X-ray 데이터셋)
- **선별 기준:** XR_WRIST(손목) 부위의 negative(정상) 케이스만 사용
- **가용 이미지:** 5,769장 (실제 건강인 손목 X-ray)

#### MURA 이미지 선별 기준 (`prepare_mura.py`)

| 기준 | 내용 |
|------|------|
| 부위 | XR_WRIST (손목) negative만 |
| 해상도 필터 | 최소 150px 이상 |
| 다양성 확보 | 동일 study당 최대 2장 (train), 최대 1장 (val) |
| 포맷 변환 | PNG/L모드 → RGB JPG (YOLO/timm 호환) |
| 재현성 | seed=42 고정 |

#### 데이터셋 최종 구성 (MURA 추가 후)

| 구분 | 골절 | 정상 (원본) | 정상 (MURA 추가) | 합계 |
|------|:----:|:-----------:|:----------------:|:----:|
| Train | 223장 | 172장 | +350장 → **522장** | 745장 |
| Val | 25장 | 22장 | +60장 → **82장** | 107장 |
| Test | 35장 | 20장 | **추가 없음** | 55장 |

> Test 세트는 오염을 막기 위해 MURA 추가 없이 원본 그대로 유지하였다.

## 4. 전처리 및 데이터 증강

### 4.1 기본 데이터 증강 (YOLO 파이프라인)

| 증강 기법 | 설정값 | 목적 |
|----------|--------|------|
| 좌우 반전 | 50% | 좌·우 손목 다양성 확보 |
| 회전 | ±15° | 촬영 각도 변화 대응 |
| 스케일 | ±40% | 확대·축소 변화 대응 |
| 이동 | ±10% | 위치 변화 대응 |
| 밝기·채도 | 소폭 | 노출 조건 변화 대응 |

### 4.2 timm 모델 학습 기법

| 기법 | 설정 | 설명 |
|------|------|------|
| SmoothedFocalLoss | gamma=2.0, smoothing=0.05 | 어려운 샘플 집중 학습, 과신뢰 억제 |
| WeightedRandomSampler | 1:1 골절:정상 | 배치 내 클래스 균형 유지 |
| Phase 1 (Frozen) | 15 에포크, backbone 동결 | 분류 헤드만 먼저 학습 |
| Phase 2 (Full) | 최대 85 에포크, 전체 가중치 | backbone lr=3e-5로 세밀 조정 |
| Composite Checkpoint | sens≥0.95 → 0.5×AUC+0.5×spec | 민감도 조건 달성 시 특이도 최대화 |

### 4.3 TTA (Test-Time Augmentation)

추론 시 8개 설정으로 예측 후 평균:

```
(hflip=False, angle=0°), (hflip=True, angle=0°),
(hflip=False, angle=+10°), (hflip=True, angle=+10°),
(hflip=False, angle=-10°), (hflip=True, angle=-10°),
(hflip=False, angle=+20°), (hflip=True, angle=-20°)
```

### 4.4 CLAHE 전처리 실험 결과

X-ray 골절선 대비 강화를 위해 CLAHE(Contrast Limited Adaptive Histogram Equalization) 전처리를 실험하였으나, 오히려 성능이 크게 저하되었다 (YOLOv8m 기준: 정확도 81.8% → 65.5%, 민감도 85.7% → 62.9%). 이후 실험에서 제외하였다.

### 5.2 최종 앙상블 아키텍처

단일 모델의 한계를 극복하기 위해 상호 보완적인 3개 모델을 앙상블하였다.

```
입력 X-ray 이미지
        │
   ┌────┴──────────────────┐
   │                       │
┌──┴──────────┐    ┌───────┴──────────────────────┐
│ YOLOv8m-cls │    │     timm CNN 계열              │
│(MURA 재학습) │    │ ┌──────────┐ ┌─────────────┐  │
│  weight=2   │    │ │DenseNet  │ │EfficientNet │  │
└──────┬───────┘    │ │   -121   │ │    -B3      │  │
       │            │ │weight=1  │ │  weight=2   │  │
       │            │ └────┬─────┘ └──────┬──────┘  │
       │            └──────┼──────────────┼──────────┘
       │                   │              │
       └───────────────────┴──────────────┘
                           │
                  가중 합산 (2:1:2) / 5
                           │
                  앙상블 골절 확률 (0~1)
                           │
                  환자·측면 단위 MAX 집계
                           │
                  임계값 0.652 적용
                           │
               골절(Fracture) / 정상(Normal)
```

#### 앙상블 가중치 설정 근거

| 모델 | 가중치 | 근거 |
|------|:------:|------|
| YOLOv8m-cls (MURA 재학습) | 2 | MURA 재학습 후 정상 평균 점수 0.503 → 0.122로 가장 큰 개선 |
| DenseNet-121 | 1 | 민감도 강점(94.3%), 특이도 낮아 낮은 가중치 |
| EfficientNet-B3 | 2 | 민감도·특이도 균형 (88.6% / 70.0%), AUC 0.850 |

## 8. 통계적 분석

### 8.1 신뢰구간 해석

환자·측면 단위 100% 성능에 대한 신뢰구간:

| 지표 | 점 추정 | 95% Wilson CI |
|------|:-------:|:-------------:|
| Sensitivity | 100% (9/9) | [70.1%, 100%] |
| Specificity | 100% (6/6) | [61.0%, 100%] |

Wilson CI의 폭이 넓은 이유: 평가 케이스 수 n=15 (골절 9명, 정상 6명)로 소규모이기 때문이다. 점 추정값은 100%이지만, 95% CI 하한선은 각각 70.1%, 61.0%이다. 대규모 외부 검증 데이터가 확보되면 더 좁은 CI를 얻을 수 있다.

### 8.2 이미지 단위 AUC 부트스트랩 CI

- 방법: 2,000회 부트스트랩 리샘플링 (seed=42)
- AUC = 0.9186 [95% CI: 0.8257, 0.9891]

### 8.3 한계 및 주의사항

| 항목 | 내용 |
|------|------|
| 소규모 평가 | 케이스 단위 n=15 → CI 폭 넓음, 외부 검증 필요 |
| 정상 레이블 | 골절 환자의 반대 손 (실제 건강인 아님) |
| 환자 일부 중복 | Train과 Test 간 동일 환자 일부 포함 가능 |
| 임계값 마진 | 최저 골절(0.686) — 최고 정상(0.643) = 0.043 (임계값 민감도 높음) |
| 단일 기관 데이터 | 다른 기관 X-ray 장비·프로토콜에 대한 일반화 미검증 |
