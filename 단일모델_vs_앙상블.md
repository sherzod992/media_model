# 단일 모델 vs 앙상블 — 정확도 비교 분석

> 주상골 골절 X-ray 자동 판독 AI | 방법론적 기여 보고서

---

## 핵심 주장

> **단일 모델은 민감도와 특이도를 동시에 달성할 수 없다.**  
> **YOLOv8m-cls + DenseNet-121 + EfficientNet-B3 가중 앙상블(2:1:2)은  
> 소규모 데이터(497장)에서도 두 지표를 동시에 100% 달성한다.**

## 1. 단일 모델의 구조적 한계

### 1.1 민감도-특이도 트레이드오프

의료 분류 모델에서 단일 모델은 민감도와 특이도를 **동시에** 높이기 어렵다.
골절을 놓치지 않으려(민감도↑) 하면 정상을 골절로 오판하고(특이도↓),
정상을 잘 걸러내려(특이도↑) 하면 골절을 놓친다(민감도↓).

```
단일 모델의 딜레마:

  민감도↑  →  모든 것을 골절로 예측  →  특이도↓
  특이도↑  →  모든 것을 정상으로 예측  →  민감도↓

  → 둘을 동시에 높이는 것이 단일 모델의 한계
```

**발견 2:** 파라미터가 클수록 좋지 않다. 497장 소규모 데이터에서는 **8~17M CNN**이 최적이다.

**발견 3:** Transformer 계열(Swin, ViT)은 소규모 의료 X-ray에서 CNN보다 열세다.

**발견 4:** CLAHE 전처리는 X-ray 골절 특징을 오히려 왜곡한다 (81.8% → 65.5%).

## 2. 앙상블 설계 원리

### 2.1 모델별 강점·약점 상호 보완

| 모델 | 강점 | 약점 | 앙상블 역할 |
|------|------|------|------------|
| **YOLOv8m-cls** | 특이도 75% (YOLO 최고) | 민감도 85.7% | 정상 판별 담당 |
| **DenseNet-121** | 민감도 94.3% (전체 최고) | 특이도 55% | 골절 탐지 담당 |
| **EfficientNet-B3** | 민감도·특이도 균형 88.6%/70% | 어느 쪽도 최고 아님 | 앙상블 안정화 |

```
DenseNet  →  "골절을 절대 놓치지 않는다" (민감도 강점)
YOLO      →  "정상을 정상으로 잘 잡는다" (특이도 강점)
EffNet    →  "두 모델 사이의 균형추"      (안정화 역할)

세 모델의 약점이 서로의 강점으로 보완됨
```

### 2.2 가중치 설정 근거

```
YOLO      : 2  ← MURA 재학습 후 정상 오탐률 0.503 → 0.122 (가장 큰 개선)
DenseNet  : 1  ← 민감도 강하나 특이도 낮아 낮은 가중치
EfficientNet: 2  ← 민감도·특이도 균형, AUC 0.850

앙상블 = (2×YOLO + 1×DenseNet + 2×EfficientNet) / 5
```

### 2.3 환자 단위 MAX 집계

```
이미지 단위 한계:
  같은 환자라도 촬영 방향(AP, Lateral, Oblique)마다 점수 다름
  → Lateral view에서 골절선 불명확 → 낮은 점수

해결:
  동일 환자·동일 측면의 여러 이미지 중 최대값(MAX)을 케이스 점수로 사용
  → 임상에서 복수 방향 종합 판단하는 실제 진료 흐름과 일치
```

&nbsp;&nbsp;&nbsp;&nbsp;→ YOLO 정상 오탐 병목 제거(0.503 → 0.122)가 최종 성능 달성의 핵심

## 5. 방법론적 기여 요약

이 연구가 증명하는 것:

### ① 모델 선택 원칙
소규모 의료 X-ray(~500장)에서는 **8~17M 파라미터 CNN**이 최적이며,
대형 모델(37M+)과 Transformer 계열은 오히려 과적합으로 성능이 저하된다.

### ② 앙상블 설계 원칙
단일 모델의 민감도·특이도 트레이드오프는 **상호 보완적 3모델 가중 앙상블**로 극복 가능하다.
모델 선택 기준: 민감도 특화 + 특이도 특화 + 균형 모델 각 1개씩 조합.

### ③ 외부 데이터 활용 전략
정상 데이터의 구조적 편향(골절 환자 반대손)은 **공개 의료 데이터셋(MURA)으로 보완** 가능하며,
이후 **재학습(YOLO fine-tuning)** 이 앙상블 병목 해소에 결정적이다.

### ④ 평가 방식
이미지 단위가 아닌 **환자·측면 단위 MAX 집계**가 실제 임상 판단과 일치하는 올바른 평가 단위다.

## 7. 결론

단일 모델은 민감도와 특이도를 동시에 달성할 수 없다는 구조적 한계가 있다.  
본 연구는 13가지 모델 실험을 통해 최적 아키텍처를 선별하고,  
**YOLOv8m-cls + DenseNet-121 + EfficientNet-B3 가중 앙상블(2:1:2)** 이  
소규모 단일 기관 데이터(497장)에서도 환자 단위 기준  
**민감도 100%, 특이도 100%, AUC 1.000** 을 달성함을 실험적으로 증명하였다.

이는 데이터 규모의 한계에도 불구하고 **모델 조합 전략과 외부 데이터 활용**이  
의료 AI 성능을 결정적으로 개선할 수 있음을 보여주는 방법론적 기여다.
